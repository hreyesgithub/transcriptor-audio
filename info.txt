### 4. Pasos para subir a GitHub y Streamlit Cloud

#### En GitHub:

1. Crea un nuevo repositorio (público) llamado `transcriptor-audio`.
2. Sube tus archivos (`app.py`, `requirements.txt`, `packages.txt`, `README.md`).

#### En Streamlit Cloud:
1. Ve a [share.streamlit.io](https://streamlit.io).
2. Conecta tu cuenta de GitHub.
3. Haz clic en **"New app"**.
4. Selecciona el repositorio, la rama (`main`) y el archivo principal (`app.py`).
5. ¡Haz clic en **Deploy**!

### 5. Otras opciones gratuitas de hosting

Si quieres probar alternativas, estas son las más destacadas:

1.  **Hugging Face Spaces:**
    *   **Pros:** Muy estable, permite usar GPUs (aunque para esta app no es necesario) y es gratuito para proyectos Open Source.
    *   **Cómo:** Creas un "Space", eliges "Streamlit" como SDK y subes los mismos archivos. No olvides que también necesitas el archivo `packages.txt` para FFmpeg.
2.  **Render:**
    *   **Pros:** Permite desplegar aplicaciones web generales.
    *   **Contra:** La versión gratuita "duerme" la app tras 15 min de inactividad y tarda un poco en despertar.
3.  **Railway:**
    *   **Pros:** Muy fácil de usar, despliegue automático desde GitHub.
    *   **Contra:** Tiene un límite de horas/créditos gratuitos al mes (aunque para pruebas suele sobrar).

**Recomendación:** Quédate con **Streamlit Cloud**; es el más sencillo y el que mejor gestiona las dependencias de audio como FFmpeg de forma nativa.
